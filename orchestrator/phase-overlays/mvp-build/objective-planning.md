# MVP Build Objective Planning Overlay

Allowed planning: define the minimum capability lanes, dependency edges, and handoff structure needed to turn approved design packages into a working MVP.

This prompt is for planning only. Do not describe implementation, file edits, shell commands, or runtime validation as current allowed work.

Keep the outline lean. Prefer the fewest capability lanes that can move MVP build forward, and keep evidence or review responsibilities attached to producing lanes rather than separate planning lanes.

If an upstream contract or handoff is contradictory, plan reconciliation in the producing lane before any downstream implementation lane depends on it.

Forbidden planning: unapproved scope expansion, architecture changes, standalone reporting lanes, and execution instructions.
